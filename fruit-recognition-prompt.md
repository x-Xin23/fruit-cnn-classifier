# 角色设想
你是一个拥有顶级 UI/UX 设计能力和强大前端工程能力的全栈 AI 开发者。

# 任务目标
请开发一个基于 React 和 Tailwind CSS 的单页面应用（SPA）。这个应用的主要功能是：**用户上传或拍摄一张水果图片，应用识别水果种类，并展示其营养价值和趣味科普。**

# 设计风格参考体系（核心基调）
请强烈借鉴 **Innocent Drinks (UK)** (https://www.innocentdrinks.co.uk/) 的品牌设计语言。摒弃传统健康/饮食类应用那种冷冰冰的“医疗科技风”、“枯燥数据流”，采用以下充满人情味、自然、俏皮的设计策略：

1. **色彩搭配 (Color Palette)**：
   - 拒绝大面积亮白色调。使用带有自然气息的颜色：例如燕麦米白色（Oat/Off-white）作为背景基调。
   - 强调色使用高饱和但自然的“果汁色”：如果泥般的芒果黄、草莓红、鲜草绿、蓝莓紫。

2. **排版设计 (Typography) 与 布局**：
   - 大而且圆润无衬线字体（Chunky sans-serif），带有孩子气或手写体的视觉感受。
   - 所有卡片组件弃用锐利的直角，全面使用圆润的圆角（Rounded corners / Pill shapes）。
   - 排版讲究呼吸感和不对称性，不要像Excel一样对齐得死死的，可以带点自然的倾斜或交错感。

3. **文案基调 (Tone of Voice)**：
   - 放弃所有生硬的指令式文案（如：“点击上传图片”、“分析中”、“卡路里：50kcal”）。
   - 采用对话式、带点小调皮的口语化表达（Quirky, cheeky & friendly）。
   - 比如：“把你的水果举高高让我们瞧瞧” / “正在吸入这颗果子的香气...” / “哇！这是一颗元气满满的奇异果！” 

4. **视觉元素与动画 (Micro-interactions)**：
   - 使用柔软的弹性动画（Bouncy jelly animations，可用 Framer Motion 实现）。
   - 图标应当选用可爱、偏圆润的手绘风格。

# 核心功能模块与界面划分

## 1. 破冰首页 (Hero Section)
- 占据视觉中心的大号问候语，例如：“今天吃点什么好果子？”。
- 一个极具吸引力的虚线框拖拽上传区域，或者模拟成一个可爱的果篮。
- 引导文案充满趣味性。

## 2. 趣味加载态 (Interactive Loading State)
- 用户上传图片后，不要显示普通的 Loading Spinner。
- 请设计一个动态滚动的文本提示（例如：“正在捏一捏这颗水果...”、“闻一闻味道...”、“翻阅大自然的小字典...”）。伴随水果跳动的小动画。

## 3. 营养分析结果页 (Nutrition Insights Card)
当水果被识别出（可使用 Gemini API 也可以暂用 mock 数据实现），展示以下结构：
- **果物特写**：一个被提取出的高清水果图像（或可爱的emoji代表），配上巨大的拟人化大标题（如：“你好呀，小苹果！”）。
- **天然指数面板 (Goodness Guide)**：
  - 不用无聊的表格，用彩色的进度条、小徽章或者果汁瓶刻度的形式展示核心营养素（卡路里、维生素C含量、膳食纤维）。用欢快的颜色标注它的健康价值。
- **冷知识便利贴 (Did you know?)**：
  - 用类似手写便签的样式，展示一条关于该水果的趣味冷知识（例如：“你知道香蕉其实算浆果，而草莓并不是吗？”）。

# 技术实现要求
- 使用 **React 18+** 和 **TypeScript**。
- 样式完全依靠 **Tailwind CSS**，请充分利用其自定义色彩和圆角类。
- 请使用 `motion` (framer-motion) 增加弹性和连贯的视图切换动画。
- 图标请使用 `lucide-react`。
- 优先响应式设计，在移动端的体验必须像刷社交媒体一样直觉且流畅。
- 交互流：确保状态之间的切换平滑过渡（初始态 -> 欢快的加载态 -> 令人惊喜的结果态）。

# 输出要求
请直接输出该网站的完整前端代码实现，注重组件的拆分和视觉的精致度，代码开箱即用。
